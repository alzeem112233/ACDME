# بنية المشروع

هذا المشروع مبني كتطبيق ويب حديث لإدارة أكاديمية رياضية.

## الطبقات الحالية

- `src/main.jsx`: واجهات الإدارة والمنطق المحلي الحالي.
- `src/styles.css`: نظام التصميم العام، RTL، والتجاوب مع الموبايل.
- `public/`: ملفات PWA والأيقونات.
- `database/schema.sql`: نموذج قاعدة البيانات المقترح للنسخة المتصلة بسيرفر.
- `dist/`: ناتج البناء النهائي للنشر، ولا يتم تعديله يدوياً.

## المرحلة الحالية

التطبيق يعمل كنسخة Local-first: البيانات تحفظ في متصفح الجهاز عبر `localStorage`.
هذا مناسب للعرض الأولي وتجربة التدفق، لكنه ليس التخزين النهائي عند التشغيل الفعلي.

## المرحلة التالية

ربط التطبيق بقاعدة بيانات وسيرفر:

1. مصادقة برقم الهاتف و OTP.
2. تخزين الأكاديميات والمستخدمين واللاعبين والدفعات في PostgreSQL أو Supabase.
3. صلاحيات حسب الدور: مالك، مدير، مدرب.
4. مزامنة offline/online للعمليات الحساسة مثل الحضور والدفعات.
